📸 사진 업로드 인프라 설정

룩북/상품 사진을 진짜로 업로드하고 표시되게 만드는 작업

🎯 작업 단계 (총 3가지)
  1. PART A: Supabase Storage 버킷 만들기 (대시보드 클릭)
  2. PART B: SQL 실행 (정책 + 컬럼)
  3. PART C: 코드 적용 + Git push

순서대로 따라하시면 30분 안에 사진 업로드 완성!

🗂 PART A: Supabase Storage 버킷 만들기

1Supabase Dashboard 접속

  1. supabase.com/dashboard 로그인
  2. chezsua 프로젝트 클릭

2Storage 메뉴 클릭

  1. 좌측 메뉴에서 Storage 클릭 (서랍 모양 아이콘)

3새 버킷 만들기

  1. 화면 좌측 또는 가운데에 "New bucket" 버튼 클릭
  2. 다음과 같이 입력:
    • Name: chezsua-images (정확히 이대로!)
    • Public bucket: ✅ 체크 (필수!)
    • 나머지는 기본값
  3. Save 또는 Create bucket 클릭
⚠️ 매우 중요
  • 버킷 이름: 반드시 chezsua-images (다른 이름이면 코드와 안 맞음)
  • Public 체크: 필수 (체크 안 하면 사진 표시 안 됨)

📋 PART B: SQL 실행

1SQL Editor 접속

  1. 좌측 메뉴 SQL Editor 클릭
  2. New query 클릭

2SQL 파일 내용 복사 → 실행

ZIP 안의 01_storage_setup.sql 내용을 복사 → SQL Editor에 붙여넣기 → Run 클릭

이 SQL이 하는 일:

  • lookbooks 테이블에 cover_image, images 컬럼 추가
  • products 테이블에 images 컬럼 추가
  • Storage 버킷에 대한 권한 정책 설정
💡 이미 컬럼이 있어도 안전

SQL에 IF NOT EXISTS 가 있어서 이미 있는 컬럼/정책은 자동 스킵해요.

4Git Push

git add .
git commit -m "Add image upload infrastructure (Supabase Storage + UI)"
git push

5Vercel 빌드 후 (1-2분) 테스트

🎯 테스트 방법

관리자 페이지에서:

  1. 관리자 로그인
  2. /admin/lookbooks/new 접속 (또는 기존 룩북 Edit)
  3. 화면 상단의 초록색 📸 Lookbook Images 영역
  4. "Drop or Click to Upload" 박스 클릭 → 사진 선택
  5. 또는 사진을 드래그&드롭
  6. 업로드 진행률 표시 → 완료!
  7. 썸네일이 격자로 표시됨
  8. 첫 번째 사진에 "Cover" 배지 자동 표시
  9. 호버하면 ←, →, × 버튼 (순서 변경/삭제)
  10. 나머지 정보 입력 → Save

📝 주의사항

업로드 실패 시 흔한 원인:
  • Storage 버킷 이름 오타 — 반드시 chezsua-images
  • Public 체크 안 함 — 다시 Storage → 버킷 클릭 → Settings에서 Public 변경
  • SQL 실행 안 됨 — Storage 정책이 없어서 권한 거부
  • 관리자 권한 없음 — users 테이블에서 role='admin' 확인
  • 파일 크기 10MB 초과 — 사진 압축 후 재시도
✅ 이 패치 완료 후 가능한 것
  • 룩북에 사진 다중 업로드
  • 드래그&드롭으로 업로드
  • 업로드 진행률 표시
  • 첫 사진이 자동으로 Cover
  • 호버해서 순서 변경 (←, →)
  • 호버해서 삭제 (×)
  • 최대 20장까지
